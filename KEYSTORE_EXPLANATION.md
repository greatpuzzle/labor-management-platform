# 키스토어 설명

## Release APK와 키스토어의 관계

### 처음 Release APK 빌드 시
1. **키스토어 생성** (한 번만)
   - `keytool` 명령어로 키스토어 파일 생성
   - 비밀번호 설정
   - 이 키스토어는 **평생 사용**합니다

2. **key.properties 파일 생성**
   - 키스토어 경로와 비밀번호 저장
   - 이 파일도 보관 필요

3. **Release APK 빌드**
   - 키스토어로 서명하여 빌드
   - `app-release.apk` 생성

### 앱 업데이트 시
- **같은 키스토어 사용 필수**
- 키스토어 없이 업데이트 불가능
- 키스토어를 잃어버리면 → 앱 업데이트 불가능

## 왜 키스토어가 필요한가?

### 1. 앱 서명 (보안)
- Android는 앱이 서명되어야 설치 가능
- 키스토어 = 앱의 "신분증"

### 2. 업데이트 인증
- 같은 키스토어로 서명된 앱만 업데이트 가능
- 다른 키스토어로 빌드하면 → "새로운 앱"으로 인식 (기존 앱 삭제 필요)

### 3. Play Store 등록
- Play Store에 앱을 등록하려면 키스토어 필수
- 한 번 등록하면 계속 같은 키스토어 사용

## 키스토어 관리 방법

### ✅ 올바른 방법
1. **키스토어 파일 백업**
   - `labor-management-release.keystore` 파일을 안전한 곳에 백업
   - USB, 클라우드 등 여러 곳에 백업

2. **비밀번호 기록**
   - 키스토어 비밀번호를 안전한 곳에 기록
   - 비밀번호 관리자 사용 권장

3. **key.properties 백업**
   - 키스토어 경로와 설정 정보도 백업

### ❌ 하지 말아야 할 것
- 키스토어 파일을 Git에 커밋 (보안 위험)
- 키스토어 비밀번호를 코드에 하드코딩
- 키스토어를 잃어버림

## 실제 사용 시나리오

### 시나리오 1: 처음 빌드
```
1. 키스토어 생성 (한 번만)
2. Release APK 빌드
3. APK 배포
```

### 시나리오 2: 앱 업데이트 (1주일 후)
```
1. 코드 수정
2. 같은 키스토어로 Release APK 빌드
3. 새 APK 배포
4. 사용자들이 새 APK 설치 → 기존 앱 업데이트됨 ✅
```

### 시나리오 3: 키스토어 분실
```
1. 키스토어 파일 없음
2. 새 키스토어로 빌드
3. 새 APK 배포
4. 사용자들이 새 APK 설치 → 기존 앱 삭제 후 새로 설치해야 함 ❌
```

## 결론

**Release APK를 사용하려면:**
- ✅ 키스토어를 **한 번 생성** (평생 사용)
- ✅ 키스토어를 **안전하게 백업** (매우 중요!)
- ✅ 업데이트할 때 **같은 키스토어 사용**

**지금은 Debug APK로 충분하지만, 나중에 Release APK를 사용할 계획이라면 키스토어를 미리 만들어두는 것이 좋습니다.**
